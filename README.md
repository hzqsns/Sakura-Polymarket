# Sakura-Polymarket

一个聚焦于 **Polymarket 研究与实践** 的仓库，用来系统整理预测市场相关的资料、观察、方法论与实验记录。

## 项目目标

这个仓库主要用于围绕 Polymarket 展开持续研究，包括但不限于：

- Polymarket 平台机制与市场结构
- 事件合约设计与结算逻辑
- 市场定价、流动性与交易行为观察
- 热门赛道与叙事跟踪
- 数据抓取、指标构建与研究框架沉淀
- 交易思路、风险控制与复盘记录

## 研究方向

后续内容计划围绕以下几类主题逐步补充：

### 1. 平台与机制

- Polymarket 的产品结构、市场创建方式与参与路径
- 市场规则、结算来源、争议处理与潜在风险点
- 与传统博彩、金融衍生品、其他预测市场产品的差异

### 2. 市场观察

- 热门事件市场的价格变化与交易节奏
- 不同题材市场之间的关注度差异
- 新闻、政策、舆论与价格波动之间的关联

### 3. 数据研究

- 市场数据抓取与清洗
- 历史价格、成交量、持仓变化等指标分析
- 发现可复用的研究模板和信号框架

### 4. 交易与策略

- 事件驱动型机会识别
- 概率定价偏差的观察与验证
- 风险收益比、仓位管理与退出机制

## 仓库规划

随着研究推进，仓库内容可以逐步扩展为如下结构：

```text
.
├── README.md
├── notes/          # 日常研究笔记
├── reports/        # 专题分析与阶段性总结
├── data/           # 原始数据与处理结果
├── scripts/        # 数据抓取、清洗、分析脚本
└── docs/           # 方法论、术语表、参考资料
```

## 使用方式

这个仓库适合作为一个长期研究基地，可以按下面的方式使用：

1. 记录你对 Polymarket 市场的日常观察。
2. 沉淀专题研究，例如美国大选、宏观事件、加密相关市场等。
3. 保存数据分析脚本与结果，逐步形成自己的研究框架。
4. 对交易判断进行复盘，优化信息收集与决策流程。

## 研究原则

- 尽量基于公开信息、可验证数据和明确规则做判断
- 区分事实、假设和主观推演
- 记录研究过程，而不仅仅记录结论
- 始终重视流动性、结算规则和事件定义带来的风险

## 说明

本仓库以研究为主，不构成任何投资建议。预测市场具有高波动、高不确定性与规则依赖性，参与前需要充分理解产品机制与风险。
